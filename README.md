## 1. サービス概要

「何から始めればいいかわからない」と感じている人が、やりたいことと現在のレベルを入力するだけで、AIが理由付きで、時間や量まで明確な「今日やるべき1つの行動」を提案するアプリです。

提案を1つに限定することで迷いをなくし、すぐに行動できる状態を作ります。

また、実行したかどうかを記録することで、行動の継続をサポートし、過去の記録を振り返ることで成長を実感できます。

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情

前職を退職した後、次の進路に迷い、やることはあるのに、何から始めればいいか決められず、結局何もできないことが多かったです。

この経験から、「何から始めればいいかわからず動けない状態」を解消し、すぐに行動できるきっかけを作れるアプリがあれば役に立つと考えました。

私は慎重な性格で、行動するまでに時間がかかることがあります。「やりたい気持ちはあるのに動けない」という状態は苦しく、自己嫌悪に陥ることもありました。　

ただ、最初の一歩さえ踏み出せれば、その後は進めることも多いと気づきました。また、 SNSや友人との会話で、「何から始めればいいかわからない」と悩んでいる人が多いことに気づきました。

## 7. このアプリで実現すること

### MVPで作る機能

**やりたいことと現在のレベルを入力 → AIによる一歩提案**

①やりたいことを入力する  
②現在の状態を選択する  
③AIが「今日やること」を1つ提案する  

・時間または量が明確なタスクにする（例：10分 / 1レッスン）  
・具体的な行動内容にする（例：「ProgateのHTML入門を1レッスン進める」）  
・その行動を提案した理由（なぜ今それをやるべきか）も提示する

### 本リリースで作る機能

**MVPに加えて、継続と振り返りをサポートする機能を追加**

① **ユーザー登録・ログイン機能**
メールアドレスとパスワードでアカウント作成・ログイン
履歴を保存できるようにする

②  提案履歴の保存・閲覧機能
 過去の提案された内容を一覧で確認できる

③ **実行チェック機能**
提案されたタスクに対して「やった / やらなかった」を記録できる

④ **振り返り機能**
過去の実行履歴をもとに、どの程度行動できたかを確認できる


## 8. このアプリの懸念点とその対策

### 懸念点①: AIの提案が適切でない可能性

**何が問題になりそうか：**
 AIが提案する「今日やること」が、ユーザーの状況に合わない可能性がある

**なぜそう思うか：**
入力情報が「やりたいこと」と「現在の状態」のみのため、ユーザーの詳細な状況までは考慮できないため  

**そのために考えている対策：**
 提案の再生成機能を実装し、合わない場合は別の提案を受け取れるようにする  
 「時間」や「量」を制限したタスクを生成することで、実行しやすさを担保する  
 プロンプト設計を調整し、具体的で行動に直結する提案を生成する

### 懸念点②：継続して使ってもらえない可能性

**何が問題になりそうか：**
一度使った後、継続して利用されない可能性がある  

**なぜそう思うか：**
毎日利用するサービスは習慣化のハードルが高く、利用動機が弱いと離脱しやすいため 

**そのために考えている対策：**
提案を「1つ」に絞ることで、毎回迷わず使える設計にする  
 実行チェック機能により、「やった」という小さな達成感を得られるようにする  
過去の提案や実行履歴を確認できるようにし、行動の積み重ねを可視化する
過去の行動履歴を確認できることで、「自分は行動できている」という実感を持たせる

## 10. 技術スタック（手段としての技術）

### 10-1. 使用予定の技術

- フレームワーク： Ruby on Rails
- DB：PostgreSQL
- デプロイ先：Heroku または Render
- 使用予定ライブラリ：
   - **OpenAI API** ：AIによる「今日できる小さな一歩」の提案生成
  - **Devise** ：ユーザー認証機能
  - **Tailwind CSS** ：UI・デザインの実装
  - **Stimulus（または Hotwire）** ：フロントエンドの動的な操作

---

### 10-2. 技術選定の理由

**なぜこの技術を使うのか**

- **Ruby on Rails を選んだ理由**
  - RUNTEQ で学んできたフレームワークで、CRUD 操作や認証機能の実装に慣れている
- **PostgreSQL を選んだ理由**
  - Rails との相性が良く、本番環境でもそのまま使える
- **OpenAI API を選んだ理由**
  - ユーザーの選択内容をもとに、自然で具体的な提案を生成するため
- **Tailwind CSS を選んだ理由**
  - シンプルで直感的な UI を素早く実装でき、レスポンシブ対応がしやすい


**今回チャレンジしたい点**

-**OpenAI API を使った AI 提案機能の実装**
  プロンプトの設計を工夫し、ユーザーに寄り添った提案を生成する
-**継続的に使ってもらうための UX 設計**
  - 「投稿して終わり」にならないよう、振り返り機能やリマインダー機能を実装し、ユーザーが「使い続けたい」と思える体験を設計する

**不安な点**
 -**OpenAI API のコスト管理**
API の使用量が増えるとコストがかかるため、リクエスト数を制限する仕組みを検討する必要がある
 
-**AI の提案精度**
 ユーザーの選択内容だけでは、細かい状況や感情まで把握できない可能性がある
