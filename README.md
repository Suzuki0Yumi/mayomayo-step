## 1. サービス概要

モヤモヤした悩みを選ぶだけで、AIが「今日できる小さな一歩」を提案してくれるアプリです。
「何からやればいいのかわからない」と感じている人が、納得して行動を起こせるようにきっかけを作ります。

---

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情

前職を退職した後、次の進路に迷い、「やることはあるのに、何から始めればいいか決められず、結局何もできない」状態に陥りました。
そんなとき、AIに相談したことで思考が整理され、自分の状態を言語化することで納得して次の一歩を踏み出すことができました。
この経験から、自分の状態と行動を整理し、「動けないとき」に一歩踏み出せるきっかけを作れるアプリがあれば、同じように悩む人の役に立てると考えました。

---

### 2-2. なぜそれが「気になった」のか

私は慎重な性格で、行動するまでに時間がかかることがあります。「行動したい気持ちはあるのに動けない」という状態は非常に苦しく、過去に自己嫌悪に陥った経験もあります。

ただ、最初の一歩さえ踏み出せれば、その後はゆっくりでも進められることがわかってきました。
そのため、友人に話すような感覚で気軽に一歩を踏み出せるアプリを作りたいと考えました。

 自分の感情や状況を言語化してもらえないので本当にこれでいいのかと不安になる


**自分でつくりたい理由**

- **選択式で迷わない**

  - 「何を書けばいいかわからない」という悩みを解消できる
  
- **1つに絞ってもらえる**
  - 「今日やること」を1つに絞ってもらえるので、迷わずに行動を始められる
  
- **自分に寄り添ってくれる**
  - 「疲れているから、軽いタスクから」といった、自分の感情や状況を整理した上で提案してくれるので納得感が生まれる

## 7. このアプリで実現すること

### MVPで作る機能

**メイン機能：モヤモヤ選択　→　AIによる一歩提案　→　履歴保存**

①モヤモヤのカテゴリを選択
選択肢から自分の状態を選ぶ
②AI提案機能
選択した内容をもとに、AIが「今日できる小さな一歩」を1つ提案する
③  提案履歴の保存・閲覧機能
 過去の提案を一覧で見られる

### 本リリースで作る機能

**MVPに加えて、継続と振り返りをサポートする機能を追加**

① **ユーザー登録・ログイン機能**
メールアドレスとパスワードでアカウント作成・ログイン
会員登録で履歴が残る

② **実行チェック機能**
提案された一歩を「やった / やらなかった」を記録できる

③ **感情の記録機能**
実行後の気持ちを簡単に記録できる（「スッキリした」「まだモヤモヤする」など）

④ **振り返り機能**
1週間、1ヶ月単位で、「どれだけ一歩を踏み出せたか」を可視化

⑤ **リマインダー機能**
毎日決まった時間に通知を送り、アプリを使う習慣をサポート


## 8. このアプリの懸念点とその対策

### 懸念点①: AIの提案が的外れだと感じられる可能性

**何が問題になりそうか：**
 AIが提案する「今日できる小さな一歩」が、ユーザーの状況や感情に合っていないと感じられる可能性がある
**なぜそう思うか：**
AIは選択肢をもとに提案を生成するが、ユーザーの細かい状況や感情までは完全に把握できない
**そのために考えている対策：**
提案の再生成機能を追加する(「この提案は合わない」と感じたら、別の提案を生成できるようにする)
 提案の前に「あなたの状況を理解しました」という共感の言葉を入れる
 提案の粒度を調整する（「5分だけ」「1つだけ」など、具体的な時間・量を示す）

### 懸念点②：継続して使ってもらえない可能性

**何が問題になりそうか：**
最初は使ってくれても、数日で使わなくなる可能性がある
**なぜそう思うか：**
 習慣化は難しく、特に「毎日使う」という行動を続けるのはハードルが高い
**そのために考えている対策：**
リマインダー機能を追加する
振り返り機能で達成感を提供する(「今日もやれた」という達成感を視覚的に表現する)

### 懸念点③：選択肢が多すぎて、逆に迷ってしまう可能性

**何が問題になりそうか：**
選択肢が多いことで、かえって使いづらくなる可能性がある
**なぜそう思うか：**
「選択式だから迷わない」というコンセプトなのに、選択肢が多すぎると本末転倒になる
**そのために考えている対策：**
初回利用時にチュートリアルを表示する
選択肢を段階的に表示する

## 10. 技術スタック（手段としての技術）

### 10-1. 使用予定の技術

- フレームワーク： Ruby on Rails
- DB：PostgreSQL
- デプロイ先：Heroku または Render
- 使用予定ライブラリ：
   - **OpenAI API** ：AIによる「今日できる小さな一歩」の提案生成
  - **Devise** ：ユーザー認証機能
  - **Tailwind CSS** ：UI・デザインの実装
  - **Stimulus（または Hotwire）** ：フロントエンドの動的な操作

---

### 10-2. 技術選定の理由

**なぜこの技術を使うのか**

- **Ruby on Rails を選んだ理由**
  - RUNTEQ で学んできたフレームワークで、CRUD 操作や認証機能の実装に慣れている
- **PostgreSQL を選んだ理由**
  - Rails との相性が良く、本番環境でもそのまま使える
- **OpenAI API を選んだ理由**
  - ユーザーの選択内容をもとに、自然で具体的な提案を生成するため  
- **Tailwind CSS を選んだ理由**
  - シンプルで直感的な UI を素早く実装でき、レスポンシブ対応がしやすい


**今回チャレンジしたい点**

-**OpenAI API を使った AI 提案機能の実装**
  プロンプトの設計を工夫し、ユーザーに寄り添った提案を生成する
-**継続的に使ってもらうための UX 設計**
  - 「投稿して終わり」にならないよう、振り返り機能やリマインダー機能を実装し、ユーザーが「使い続けたい」と思える体験を設計する

**不安な点**
 -**OpenAI API のコスト管理**
API の使用量が増えるとコストがかかるため、リクエスト数を制限する仕組みを検討する必要がある
  
-**AI の提案精度**
 ユーザーの選択内容だけでは、細かい状況や感情まで把握できない可能性がある
